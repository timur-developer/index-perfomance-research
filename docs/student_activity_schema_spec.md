# Текстовое ТЗ для рисунка схемы `student_activity`

Этот файл нужен как воспроизводимое описание рисунка схемы данных, который затем можно нарисовать в любом удобном редакторе.

## Что должно быть на рисунке

- Один главный блок с названием `student_activity`.
- Внутри блока перечислить все поля таблицы.
- Поля желательно сгруппировать по смыслу.
- Индексируемые поля отметить отдельным визуальным маркером.

## Поля

### Идентификаторы

- `activity_id`
- `student_id`
- `course_id`

### Организационные поля

- `faculty`
- `group_code`
- `semester`

### Учебная активность

- `activity_type`
- `status`
- `score`

### Временные поля

- `created_at`
- `updated_at`

## Какие поля нужно отметить как индексируемые

- `student_id`
- `created_at`
- `status`
- составная пара `status + created_at`
- составная пара `course_id + score`

## Рекомендуемая подпись к рисунку

```text
Структура набора данных student_activity и поля, использованные для исследования индексов
```
